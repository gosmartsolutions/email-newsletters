Email Sending API

Email Sending Using SendGrid & Mailgun API's

To send emails you need to run the script at send_newsletter.php in the root of your directory or subfolder. Example: yourdomain.com/email_send/send_newsletter.php would be how to run it if you installed this in an email_send sub directory on your main domain. Also load the sql/email-newsletter-tables.sql file into a MySQL database and make sure and change the settings in applications/Config.php. Users you want to email should be stored in the users table and the emails you want to send should be stored in the email_templates table. Both of these tables need to be populated in order for this program to work out of the box.

Installation

You need PHP 5+, MySQL and Composer installed. You also need PDO enabled on the server. 1.) Import the sql/email-newsletter-tables.sql file into your MySQL database, 2.) Change the settings in applications/Config.php, 3.) Upload all files to the web directory on your domain where you want this to reside.

Install Composer

In order for this to work out of the box you will need Composer installed on your server. Read the Composer installation instructions for how to do this.

Using the send_newsletter.php script

By default the script will pull the default paramaters that you have set in your applications/Config.php file. You can override these defaults with querystring values for server and limit. Example: send_newsletter.php?server=sendgrid&limit=100 will send the newsletter using sendgrid and will limit the send to 100

Add email templates

This script does not have a web interface for adding email templates but you can add email templates directly in the "email_templates" database table . Or you can hire us to create an email templates management web interface.

Email send log

The system keeps track of all emails sent per email template in the sent_emails table. Anytime you run send_newsletter.php, it references this table to make sure it does not resend to the same email addresses for the same email template.

Open and click tracking

Open and click tracking are stored in the email_tracking table. URLs in your template are automatically re-written in order for tracking to work. You need to include the {open_bug} tag at the bottom of your template if you want to record opens.

Opt Out

This script DOES NOT include any opt out features or forms. You will need to create your own opt out form that updates the users table when someone opts out and include a link to it at the bottom of the template. Or you can hire us to create an opt out form for you.

Upload Users & Emails

This script does not have an interface for uploading email addresses but you can do this directly in the database by uploading your file to the users table. Or you can hire us to create an email upload web interface.
